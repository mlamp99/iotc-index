# /IOTCONNECT® Enablement Hub

A static, marketing-grade catalog of the Avnet [/IOTCONNECT](https://www.avnet.com/iotconnect)
ecosystem — **pre-enabled development boards, SDKs, and ready-to-run demos** — browsable
by partner, board, topic and language, with per-board QuickStart guides, demos, videos
and buy links.

- **Production:** https://avnet-iotconnect.github.io/iotc-index/
- **Staging (this repo):** https://mlamp99.github.io/iotc-index/
- **Hosting:** GitHub Pages, served straight from `main`

The entire site is generated from a few **CSV files**. You edit the CSVs,
commit them, and a GitHub Action rebuilds the data the page renders.

---

## What the site does

- **Hero + partner showcase** — Avnet-branded landing with live counts and the five
  silicon partners (Infineon · Microchip · NXP · Renesas · STMicroelectronics).
- **Hardware view** — every enabled dev board as a card (image, vendor, part number,
  AWS/Greengrass badges, project count). Click any board to open a **detail drawer**
  with where-to-buy, QuickStart & Developer guides, demos, webinars, videos and blogs.
- **SDKs & Demos view** — the open-source repos across the org (SDKs, libraries, example
  projects) with language, topic and status filters.
- **By-partner view** — everything grouped by silicon vendor.
- Full-text search, deep-links (see below), keyboard-accessible cards/drawer, and a
  robust image fallback (Azure host → bundled local image → monogram).

## Sharing links

The site is a single page, so you can deep-link to a filtered/expanded state by
appending to the URL. The address bar also updates as you click around, so you can
just navigate to what you want and copy the link.

| To share… | Append | Example |
|---|---|---|
| One **manufacturer** | `#partner/<slug>` (or `?partner=<slug>`) | `…/#partner/st` · `…/#partner/nxp` |
| A specific **board** | `#board/<slug>` | `…/#board/aes-rzb-v2l-sk-g` |
| A **view** | `#hardware` · `#software` · `#maker` | `…/#software` |

Manufacturer resolves by name, slug, or alias — `st`, `nxp`, `infineon`, `microchip`,
`renesas`, plus any other vendor in the catalog (`advantech`, `amd`, `nordic`,
`raspberry-pi`, `tria-nxp`, …). `st` is a shortcut for `stmicroelectronics`.

## How it works (data flow)

```
data/*.csv  (listings · boards · resources · config)   ← source of truth (you edit these)
          │
          │  build_site.py  +  live GitHub facts (stars, last-updated)
          ▼
   index.json  +  AUDIT.md  ← generated artifacts (do not hand-edit)
          │
          ▼
      index.html            ← Avnet-branded front end; fetches index.json in the browser
          │
          ▼
   GitHub Pages (live site)
```

The data lives in **CSV** (not a binary spreadsheet) so GitHub renders each file as a
table and shows row-level diffs in commits and PRs. Edit the CSVs in LibreOffice,
Google Sheets, or VS Code — see [`data/README.md`](data/README.md).

`.github/workflows/build-index.yml` runs `build_site.py` on every commit under `data/`
(and weekly) and commits the refreshed `index.json` + `AUDIT.md`. `index.html` is never
touched by CI.

## The catalog (`data/*.csv`)

Four CSVs — **`listings`**, **`boards`**, **`resources`**, **`config`** — each one
editable in any spreadsheet tool and fully diffable on GitHub. Column-by-column docs
live in **[`data/README.md`](data/README.md)**. In short:

- **`listings.csv`** — the SDK/demo cards. A `sample` with several boards expands to one
  card per board; an `sdk`/`library` stays one card. Board-less rows take an `Image`
  (tech logo / photo / URL) so every card has a picture.
- **`boards.csv`** — the hardware registry, with `Image File` (Azure) + `Image Local`
  (offline fallback) and AWS/Greengrass qualification flags.
- **`resources.csv`** — per-board Buy / QuickStart / Developer / Demo / Webinar / Video /
  Blog links that power the board detail drawer.
- **`config.csv`** — `ORG`, `IMAGE_BASE`, `IMAGE_LOCAL_BASE`, `BRAND_BASE`,
  `DEFAULT_STATUS`, `PAGES_URL`.

---

## Board images

Each board can resolve an image from three layers, in order:
1. **`Image File`** at the Azure `IMAGE_BASE` host (the curated photo set), or a full URL.
2. **`Image Local`** committed under `assets/boards/` (offline-safe; future-proof for the
   move into the avnet repo).
3. A clean vendor **monogram** placeholder if neither loads.

This makes the grid render reliably even if a remote host is down or a file is missing.

## Building locally

```bash
python build_site.py            # reads data/*.csv, writes index.json + AUDIT.md (no deps)
python -m http.server 8000      # then open http://localhost:8000/
```

Optional env: `GITHUB_TOKEN` (live stars/last-updated + uncatalogued-repo audit),
`CATALOG_DIR` (default `data`), `OUT_DIR`, `ORG`.

To re-audit link liveness: `python migration/_linkcheck.py`.

---

## `AUDIT.md` — the gap report

Regenerated on every build. Surfaces: boards referenced but undefined, boards with no
image, orphan boards, org repos with no listing (needs `GITHUB_TOKEN`), and listings
missing description/topics. The header reports live counts.

## Common tasks

- **Add/edit a board or listing** → edit `data/boards.csv` / `data/listings.csv`, commit
  (CI rebuilds), or run `python build_site.py` locally and commit the regenerated
  `index.json`/`AUDIT.md`.
- **Add board guides/demos/buy links** → add rows to `data/resources.csv` keyed by the
  board's Part Number.
- **Add a board image** → drop the file in `assets/boards/` and set `Image Local` (and/or
  upload to the Azure host and set `Image File`).
- **Hide something without deleting** → set `Include = no`.
- **Check what's incomplete** → read `AUDIT.md`.
